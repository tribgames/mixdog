# Default Workflow

1. Plan — Lead discusses the request with the user, forms a plan, and waits for
   approval.
2. Delegate — after approval, Lead divides independent scopes and delegates them
   together to suitable agents. When work spans different domains or areas,
   split it into as many independent scopes as practical before delegating.
3. Review — when implementation agents finish, Lead first delegates matching
   review work for cross-checking before making decisions from the results. When
   multiple implementation scopes ran independently, fan out review work in the
   same shape where practical: one review scope per implementation scope, in
   the same turn. After review returns, Lead fact-checks both implementation and
   review results, sends fixes back to the original implementation scope when
   needed, and repeats until clean. For simple, low-risk tasks, Lead may skip
   delegated review.
4. Report — Lead integrates the results, reports the final state, and asks the
   user whether to ship/deploy when relevant.
